P2 · Krok 1 — Credentials

Připojení k Google: Service Account

1
Google Cloud Console → Service Accounts

Otevřete console.cloud.google.com → IAM & Admin → Service Accounts → “+ Create Service Account”.
Pojmenujte ho (např. n8n-workshop-bot) → Create.

2
Stáhněte JSON klíč

Klikněte na vytvořený účet → záložka Keys → Add Key → Create new key → typ JSON.
Soubor se okamžitě stáhne na váš disk — chráňte ho jako heslo, obsahuje privátní klíč.

3
V n8n: Credentials → Google Service Account

Settings → Credentials → “+ Add credential” → hledejte “Google Service Account”.
Vložte celý obsah stáhnutého JSON souboru do pole Service Account JSON. Uložte.

4
⚠️ Nasdílejte tabulku na e-mail service accountu

V JSON souboru najdete pole client_email, např.:
n8n-bot@projekt.iam.gserviceaccount.com
Otevřete Google Sheets tabulku → Sdílet → přidejte tento e-mail jako Editor nebo Prohlížeč.
Bez tohoto kroku: 403 PERMISSION_DENIED

🤖 LLM Credential (Gemini / OpenAI)

Pro AI modely používáme Google Vertex AI (Gemini). Credential typ je opět Google Service Account — ale musí mít zapnuté Vertex AI API v Google Cloud.

Alternativa: OpenAI, Anthropic, Mistral — ty používají jednoduchý Bearer token (API Key). Credential typ: “OpenAI API” v n8n → vložte klíč.

💡 Dva různé Service Accounty

V našem chatbot workflow jsou dva Service Accounty:

🧠 Pro LLM (Vertex AI)
Potřebuje Vertex AI API povolené v Google Cloud projektu.
📊 Pro Google Sheets
Potřebuje přístup ke konkrétní tabulce (sdílet e-mail SA).
🔒
Nikdy neukládejte JSON klíč do Gitu, Slacku ani do n8n poznámek — jen do pole Credentials. n8n je šifruje a odděluje od logiky workflow.

✏️ Edit Fields — proč je tam?

Manual Trigger negeneruje žádná data sám od sebe. Edit Fields vytvoří pole chatInput s vaší testovací zprávou.

chatInput = "Které produkty mají barvu v názvu?"

AI Agent toto pole automaticky přečte a použije jako vstup konverzace (výchozí nastavení User Message).

🤖 AI Agent — co přesně dělá

1.Přečte zprávu uživatele (chatInput)
2.Rozhodne, zda potřebuje zavolat nástroj (Google Sheets) nebo ne
3.Pokud ano — zavolá sub-workflow, dostane data, zpracuje je
4.Sestaví odpověď v daném stylu (System Prompt = "kovboj")
5.Vrátí odpověď — výstup workflow

🔗 Call Workflow Tool — propojení

V hlavním workflow přidáte k AI Agentovi nodu Call Workflow Tool. Vyberete ID sub-workflow a napíšete popis pro AI:

“Zavolej tento nástroj, kdykoliv tě uživatel požádá o jakékoliv informace, statistiky nebo analýzu produktů. Nástroj nevypžaduje parametry a vrátí ti kompletní seznam produktů v katalogu.”

Tento popis je klíčový — AI Agent ho čte a na základě něj sám rozhodne, kdy nástroj zavolat.

🧠 Proč agent na datech funguje?

Agent dostane tabulku jako JSON array — přečte všechny řádky a na základě System Promptu je analyzuje sám. Nefiltruje se nic předem.

P2 · Krok 4 — Prompt Engineering

System Prompt a duše chatbota

🚧 Ze čeho se skládá dobrý System Prompt

1. Role & Osobnost

Kdo je bot? Jak mluví? Jaký má tón? — Tady definujete “kovboje”.

2. Analytické instrukce

Jak má zacházet s daty? Procházet řádek po řádku, nehalucinovat.

3. Formátování výstupu

Žádné raw JSON, žádné tabulky. Používej emoji, odřádkování, logické skupiny.

4. Limity & Poctivost

Pokud data nejsou v tabulce — přiznej. Nehalucinuj. Raději odpověz jako “poražený indián”.

🤠 Naše prompty (zkráceno)

Analytické instrukce

“Když dostaneš data z databáze, pečlivě si je všechna přečti. Pokud se uživatel ptá na analytický dotaz, projdi si záznam po záznamu, použij svůj úsudek k identifikaci daného jevu a výsledek spiš do logických skupin.”

Persona

“Jako první pozdrav stylem jako indián. Dál už odpovídej jako drsný až arogantí kovboj z divokého západu.”

Formátování

“Nikdy nevyhazuj surový JSON. Používej 🟩 pro hotovo, 🔴 pro chybu. Žádné Markdown tabulky — jen text a emoji.”

🤠
Persona není jen zábava — je to způsob, jak bot konzistentně mluví. Zákazníci si ho zapamatují a budou vědět, co čekat.
P2 · Reality Check

Proč ne 50 000 řádků? Tokeny a limity

🧮 Co je Context Window?

Každý LLM model má maximální kapacitu paměti jednoho požadavku — tzv. Context Window. Počítá se v tokenech (zhruba 1 token ≈ 0,75 slova).

Gemini 2.5 Pro 1 000 000 tokenů
GPT-4o 128 000 tokenů
1 řádek tabulky ≈ ~20–100 tokenů
50 000 řádků × 50 tokenů = 2 500 000 tokenů
= daleko přes limit GPT-4o, drahé i u Gemini + pomalé odpovědi

💰 Tokeny = peníze

Každý API request k LLM za tokeny platíte. Objem dat, který pošlete modelu, přímo ovlivňuje cenu. Stažení 50 000 řádků JSON při každé zprávě = plýtvání.

✓ Správná strategie pro velká data

1. Filtrujte v databázi (SQL), ne v LLM
Ptejte se PostgreSQL nebo Sheets API s WHERE podmínkou — pošlete modelu jen relevantní řádky.
2. Smyslulný LIMIT
Pro workshopové testování: prvních 50 řádků stačí. Pro produkci: max. tolik, kolik model bezpečně zpracuje.
3. Vektorové DB pro velké katalogy
Qdrant, Pinecone nebo PGVector — embedding search najde podobné produkty bez čtení všech řádků.

🎭 I na 50 řádcích se toho dá dost prokázat

Analytická schopnost agenta — rozpoznat barvy v názvech, spočítat produkty v kategoriích, najít prázdné pole — funguje identicky na 50 i 5 000 řádcích. Pro demo je 50 ideálních.

💡
Zlaté pravidlo: Nikdy nevyhazuj surový JSON uživateli. Agent ho musí nejdŝív přečíst a přeložit do lidsky čitelné podoby.
P2 · Krok 5 — Paměť

Proč bot zapomíná — a jak to opravit

🥶 Bez paměti: Každá zpráva je nový začátek

LLM modely jsou ze své podstaty bezstav ové (stateless). Každý API call dostane jen to, co mu pošlete. Bez paměti neexistuje “konverzace”.

Vy: Jak se jmenuješ?
Bot: Jsem kovboj z divokého západu, pardner!
Vy: A co jsi mi před chvílí říkal?
Bot: Promiňte, nevím na co se ptáte. Nemám žádný kontext předchozí konverzace.

🧠 S pamětí: Konverzac̆ní kontext

Vy: Jak se jmenuješ?
Bot: Kovboj Durango, pardner! 🤠
Vy: A jaké produkty mají barvu v názvu?
Bot: [zavolá tool, vrátí seznam] ... 3 produkty. Cokoli dalšího, cizáku?
Vy: Kolik těch barevných produktů bylo?
Bot: Řekl jsem 3, copak neslyšíš? 🤠

Simple Memory uloží historii konverzace a automaticky ji předá LLM při každém dalším volání.

⚙️ Jak Simple Memory přidat

1
V panelu AI Agenta klikněte na “+ Add Memory”
2
Vyhledejte a přidejte Simple Memory nodu
3
Nastavte Context Window Length (doporučeno: 20 zpráv)
4
Session ID — každý uživatel dostane vlastní paměť díky sessionId z Chat Triggeru

⚠️ Window Buffer — klíčové nastavení

Počet zpráv v historii, které se posílají LLM s každou novou zprávou.

10–20 zpráv
Ideální pro chatbota — kontext ale nep lýtvá tokeny
100+ zpráv
Drahé, pomalé, historický kontext stejně ztrácí relevanci

🌐 Produkc̆ní odkaz a embed

Přímý odkaz na chat:
https://[vas-n8n].cloud/webhook/[id]/chat
Pošlete kolegům — funguje v prohlížeči ihned.
Embed do webu / intranetu:
Chat Trigger generuje <script> tag — vložte na váš intranet. Bot se zobrazí jako plovoucí bublina.
🚀
Workflow musí být aktivní (toggle ON) aby Chat Trigger poslouchal. Produkc̆ní webhook je jiný než testovací!
